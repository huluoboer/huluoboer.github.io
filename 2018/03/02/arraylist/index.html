1.流的基本概念

1.1什么是流？

流是Java8 引入的全新概念，用来处理集合中的数据，暂时可以理解为一种优雅 的集合处理方式
传统对集合的操作相对麻烦。流的声明式写法，链式操作，代码简洁，可读性更高（个人认为）

###1.2流的特点
1.流只能遍历一次
流水线的概念，一旦一个产品完成流的操作，便不能再次拿到并处理。
2.采用内部迭代的方式
声明式由流自行处理内部，我们只需表达。

###1.3流的操作过程
1.数据源
2.执行中间操作–>流水线操作
3.终端操作—>本次流操作结束

###1.4 Optional介绍
Optional是Java8新加入的一个容器，这个容器只存0个或1个元素，它用于防止出现NullPointException
API:
isPresent()
判断容器是否有值
ifPresent(lambda) 容器若不为空执行则 Lambda
T get()
若容器为空 则抛出NoSuchElement异常
T orElse(T else)
获取容器中的元素，若容器为空则返回括号中的值

###2.13归约
归约是将集合中的所有元素经指定运算，折叠成一个元素输出，如：求和，求平均值
reduce函数实现归约

int age=list.stream.reduce(0,(p1,p2)->p1.getAge()+p2.getAge());

reduce的第一个参数表示初始值
第二个参数需要进行的操作，他将结合艘带有两个参数的lambda的表达式

###2.2收集器的简介
收集器将经过筛选，映射的流进行最后的 整理，可以 使得最后的结果以不同的结果展示。
collect方法即为收集器，他接收Collector接口实现作为 收集器的收集方法
Collect接口提供了很多默认的实现方法，也可以自定义实现
静态方法的使用：

###2.2.1计数
list.stream.collect(Collectors.counting)
list.stream.collect(Collectors.maxBy(Comparator.comparingInt(Person::getAge)))
list.stream.collect(Collectors.summingInt(Person::getAge))

###2.2.2 连接字符串
list.stream.collect(Collectors.join())

###2.3 一般性的规约操作
Collectors.reducing(1,2,,3)
初始值
操作的字段
操作过程
Optional sumAge = list.stream()
.collect(Collectors.reducing(0,Person::getAge,(i,j)->i+j));

###2.4 一级分组
list.stream.collect(Collectors.groupingBy((person)->{ ... }))

###2.4 二级分组
两个参数重载

.collect(Collectors.groupingby((person)->{

}, groupingby(Person::getSex)));
